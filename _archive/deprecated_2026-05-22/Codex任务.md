# Codex任务.md

## 用途

这是给 Codex 使用的任务池。它把当前任务分成三类：当前必须做、等 ChatGPT 确认后再做、暂时不要做。

## 当前必须做

| 优先级 | 任务 | 说明 |
| --- | --- | --- |
| P0 | 维护共享入口文件 | 确保 `新对话从这里开始.md`、`项目概览_英文旧版.md`、`CHATGPT_MEMORY_SYNC.md` 可读且最新。 |
| P0 | 保持同步文件更新 | 每次完成任务后更新 `同步包.md`、`任务看板.md`、`变更记录.md`。 |
| P0 | 保护已确认设定 | 不擅自改动角色、世界观、开局结构和风格方向。 |

## 等 ChatGPT 确认后再做

| 优先级 | 任务 | 需要 ChatGPT / 用户确认的内容 |
| --- | --- | --- |
| P0 | 编写第一集完整剧情大纲 | 需要确认第一集结构、关键情绪节点、结尾钩子。 |
| P0 | 设计第 2-4 集角色登场 | 需要确认陆衡、林见舟、许明烬登场顺序和每集主事件。 |
| P0 | 完善第 5 集《无人到站》 | 需要确认单集结构、主角团分工、情绪回收对象。 |
| P1 | 建立第一集关键帧 Prompt | 需要先有第一集大纲和关键场景列表。 |
| P1 | 更新陆衡新版角色海报资产 | 需要用户确认视觉方向和参考图资产。 |

## 暂时不要做

| 事项 | 原因 |
| --- | --- |
| 大幅改写核心世界观 | 当前世界观已确认，改动会造成记忆漂移。 |
| 把角色能力游戏职业化 | 与成熟科幻情感悬疑剧方向冲突。 |
| 擅自创建大量新角色和组织 | 会稀释当前主角团与静海系统主线。 |
| 直接写完整长剧本 | 当前更需要分集大纲、登场节奏和验收标准。 |
| 删除中文源文件 | 这些文件是当前项目设定的主要来源。 |

## Codex 每次任务结束必须输出

1. 改了哪些文件。
2. 为什么改。
3. 如何验证。
4. 下一步需要 ChatGPT 或用户确认什么。
